# CRM Relational Database Design & Analysis 📊💼

## 📌 Introduction
Customer Relationship Management (CRM) systems are the backbone of modern business operations, enabling organizations to effectively understand, track, and manage customer interactions. This project delivers a comprehensive relational database solution built specifically for CRM operations. It covers the full data lifecycle—from architectural database modeling and schema creation to populating structured sample data and executing advanced analytical SQL queries. Designed with performance and scalabilty in mind, this database provides action-oriented insights to drive data-informed decision-making.

## 🔍 File Details & Descriptions

### 1. `ERD_Diagram`
A visual representation of the Entity-Relationship Diagram (ERD). It models the conceptual database architecture, outlining core entities, their attributes, primary/foreign key relationships, and overall structural cardinalities.

### 2. `Mapping_Diagram`
Illustrates the logical schema mapping, detailing how the conceptual ERD translates into relational database tables, data types, constraints, foreign key placements, and normalization rules.

### 3. `Implementation.sql`
Contains the complete SQL DDL and DML scripts required to physically build and populate the database. It includes table creation, relationship/constraint configurations, and initial data insertion for an operational environment.

### 4. `data_analysis.sql`
Features a comprehensive collection of analytical SQL queries designed to extract business insights. It includes complex multi-table JOINs, aggregations, performance metrics, and data segmentation to drive decision-making.

---

## 🚀 How to Get Started
1. Clone this repository to your local machine.
2. Execute `Implementation.sql` in your SQL environment to build the relational structure.
3. Open and execute `data_analysis.sql` to review analytical queries and insights.
